## Setup

No installation required. Obsidian reads the wiki directory as a vault.

To open the wiki in Obsidian:
1. Open Obsidian
2. "Open folder as vault" → select the Seojae project root
3. Install recommended community plugins (optional):
   - **Graph View** (built-in) — visualize wiki page connections
   - **Dataview** — query wiki pages by frontmatter fields

## Workflows

### Wikilink Conventions

All wiki pages use Obsidian-compatible wikilinks:
- Link format: `[[Page Name]]`
- Use one canonical name per concept (always `[[Attention Mechanism]]`, not `[[어텐션 메커니즘]]`)
- Alternative names go in frontmatter `aliases` field
- Filenames are always English kebab-case (e.g., `attention-mechanism.md`)
- Filenames must be unique across all wiki/ subdirectories (Obsidian "shortest path" compatibility)

### Git Integration

If using Obsidian Git plugin for auto-backup:
- Commit convention: `auto: obsidian backup <date>`
- This convention is recognized by the wiki schema but not generated by it

### .obsidian/ Directory

The repo's `.gitignore` ignores `.obsidian/` entirely by default. If you want to
share Obsidian plugin configuration across devices, selectively un-ignore:

```gitignore
# In .gitignore — replace the blanket ignore with selective tracking:
# .obsidian/           ← comment out or remove
.obsidian/workspace*.json
.obsidian/graph.json
```

This tracks plugin configs while still ignoring device-specific workspace state.

## Configuration

| Option | Default | Description |

| Shortest path links | Enabled | Obsidian resolves `[[Page]]` across subdirectories |
