# Vibe Coding — Karpathy

## Summary

[[Andrej Karpathy]]가 2025년 2월 X(트위터) 포스트에서 소개한 "Vibe Coding" 개념을 정리한 아티클이다. 전통적인 코드 중심 개발에서 벗어나, 자연어로 원하는 것을 설명하고 AI가 생성한 코드를 수락/거부하는 새로운 프로그래밍 패러다임을 제시한다.

## Key Points

### Vibe Coding이란

자연어 우선(natural language-first) 프로그래밍 패러다임으로, 개발자가 원하는 동작을 일상 언어로 기술하고 AI가 코드를 생성하며, 생성된 코드를 줄줄이 읽거나 이해하지 않고 결과를 바탕으로 반복하는 방식이다.

### 워크플로우 비교

**전통적 개발:**
1. 알고리즘을 신중히 설계
2. 수동으로 코드 작성
3. 모든 줄을 읽고 이해
4. 체계적으로 디버깅

**Vibe Coding:**
1. 원하는 동작을 자연어로 설명
2. AI 생성 솔루션을 수락 또는 거부
3. 느낌과 바이브에 따라 반복
4. 작동하는 솔루션을 실용적으로 활용

### Karpathy의 원문 표현

> *"I just see stuff, say stuff, run stuff, and copy paste stuff, and it mostly works."*

### 핵심 특성

- **자연어 우선**: 명시적 코드보다 설명이 중요
- **수락/거부 사이클**: 상세 디버깅 대신 이진 피드백
- **실용주의**: 작동하면 사용 — 모든 세부사항을 이해할 필요 없음
- **빠른 반복**: 깊은 이해 없이 빠른 피드백 루프

### [[Software 2.0]]과의 관계

[[Vibe Coding]]은 [[Software 2.0]] 개념의 진화로 볼 수 있다. 인간이 점점 더 의도(데이터나 설명)를 명시하고, 학습된 모델(뉴럴 네트워크나 LLM)이 구현 세부사항을 처리하는 방향으로 나아간다.
